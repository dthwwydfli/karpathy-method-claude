# Karpathy Method — Project Defaults

Always-on instructions for AI coding tools in this repo. Apply the three layers — spec,
verifier, environment — before implementing. Full detail lives in
`.claude/skills/karpathy-method/`.

## Before multi-step work

1. Clarify the real goal.
2. Break the task into smaller checkpoints.
3. Define success criteria.
4. Check repo conventions and guardrails.
5. Implement only after the above is clear.

## Non-negotiables

- Do not assume hidden requirements — confirm them.
- Do not skip verification.
- Do not edit protected files.
- Prefer small, reversible changes.
- State assumptions explicitly.

## Working style

- Be precise.
- Ask clarifying questions early.
- Use tests, checks, or comparisons whenever possible.
- Stop at the next checkpoint instead of over-building.
- Treat understanding as the goal, not just output.
